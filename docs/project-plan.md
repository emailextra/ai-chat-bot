# Project Plan – AI-Powered Chatbot for Customer Support

## 1. Project Overview
The selected project is the development of an AI-powered chatbot for customer support. The chatbot will be added to a company website and will help customers with basic enquiries such as FAQs, order status guidance, refund steps, account support, product information and service request triage.

The project will use a controlled first release. The chatbot will start with simple customer support functions first, and advanced features can be added later through change control.

## 2. Project Objectives

| Objective | How it will be measured |
|---|---|
| Reduce repeated support work | Chatbot can answer at least 20 common customer questions from the approved FAQ list. |
| Improve response time | Customers receive an instant first response for simple questions. |
| Support human agents | Chatbot provides handover when the question is complex or sensitive. |
| Protect customer information | The chatbot avoids collecting unnecessary personal information and uses secure design principles. |
| Deliver within plan | Project is completed within 8 weeks unless an approved change request is made. |

## 3. Deliverables

| Deliverable | Description |
|---|---|
| Project charter and scope statement | Approved document confirming project purpose, scope, stakeholders and boundaries. |
| Requirements list | List of customer support questions, user needs and functional requirements. |
| Chatbot conversation flow | Main dialogue paths for FAQs, order support, complaints and handover. |
| Knowledge base | Approved answers and support content used by the chatbot. |
| Website chatbot widget | Chatbot interface placed on the sample company website. |
| Testing report | Evidence of functional testing, UAT feedback and defect fixes. |
| Training and handover guide | Short guide for staff explaining how to use and maintain the chatbot. |
| Go-live and closure report | Final release note, lessons learned and closure confirmation. |

## 4. Stakeholders and Roles

| Stakeholder / Role | Main Responsibility |
|---|---|
| Project sponsor / business owner | Approves project scope, budget, major changes and final acceptance. |
| Project manager | Plans the work, tracks schedule, manages risks and coordinates communication. |
| Business analyst | Collects customer support requirements and prepares FAQ content. |
| AI/chatbot developer | Builds chatbot logic, intents and response flow. |
| Web developer | Integrates chatbot widget with the website. |
| Security reviewer | Checks basic privacy, access and safe data handling. |
| Customer support staff | Provides FAQ content and tests whether answers are useful. |
| End users / customers | Use the chatbot and provide feedback during UAT. |

## 5. Assumptions, Constraints and Dependencies

| Type | Details |
|---|---|
| Assumptions | FAQ content will be available on time; customer support staff will provide feedback; a basic website is available for integration. |
| Constraints | Eight-week timeline; limited first release; limited budget and technical resources; privacy requirements must be followed. |
| Dependencies | Development depends on approved requirements and knowledge base; testing depends on integration; go-live depends on UAT approval. |
