# Change Request Log – AI-Powered Chatbot for Customer Support

## Change Request Process

A change request process is needed because new chatbot features can easily increase the project scope. Only approved changes will be added to the scope, schedule or deliverables.

| Step | Description |
|---|---|
| 1. Raise request | A team member or stakeholder submits the change request with reason and expected benefit. |
| 2. Record request | The project manager records it in the change request log. |
| 3. Impact analysis | Team checks impact on time, cost, quality, privacy, testing and resources. |
| 4. Decision | Sponsor/project manager approves, rejects or defers the request. |
| 5. Update plan | If approved, scope, schedule, risk register and task list are updated. |
| 6. Implement change | Assigned owner completes the approved change. |
| 7. Review and close | Team checks the result and closes the request. |
